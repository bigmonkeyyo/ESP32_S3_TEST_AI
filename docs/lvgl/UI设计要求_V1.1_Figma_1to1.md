# UI设计要求_V1.1_Figma_1to1

## 1. 目标与范围
- 目标分辨率：`320 x 240`（横屏）
- 目标：严格按 Figma `LVGL_320x240_UI_Home_Settings_Status` 1:1 还原
- 页面范围：
  - `Home Screen 320x240`（节点 `4:2`）
  - `Settings Screen 320x240`（节点 `4:24`）
  - `Status Screen 320x240`（节点 `4:69`）
  - `WiFi Setup Screen 320x240`（节点 `4:118`）

## 2. 强制流程（必须遵守）
1. 先在 LVGL 模拟器开发与调试。
2. 每个页面完成后，导出模拟器截图，与 Figma 逐项对比（尺寸、颜色、字号、间距、圆角、边框、文字内容）。
3. 对比通过后，再移植到 ESP32 实际工程页面框架。
4. 完成编译、烧录、联调验证通过后才算任务完成。

## 3. 中文字体规范（强制）
- 禁止使用 LVGL 自带中文字体（如 `lv_font_simsun_16_cjk`）。
- 必须使用外部工具生成项目专用中文字库 C 文件并引用。
- 推荐字库：`Noto Sans SC`（Regular/Medium/Bold）。
- 字库生成原则：
  - 按页面实际用字建立字符子集，避免全量字库。
  - 按字号与字重拆分多个字体文件，禁止“一个大而全字体”。
  - 所有字体 C 文件纳入项目版本管理。

### 3.1 建议字体切片（与设计稿对齐）
- `30/Bold`：时间 `14:28`
- `24/Bold`：天气主温度文案
- `18/Bold`：页面主标题（设置/设备状态/WiFi连接）
- `14/Medium`：主按钮文案
- `13/Regular`：列表左侧标签、返回文案、顶部城市天气等
- `12/Regular/Medium`：状态值、提示文案、次级信息
- `11/Regular/Medium`：胶囊标签、底部提示、小字

## 4. 全局视觉规范
- 页面容器：
  - 外框：`1px #D7E7F8`
  - 圆角：`14`
  - 背景渐变：`#F2F9FF -> #FFF6EA`（横向）
  - 内边距：`12`
  - 页面内主垂直间距：`8`
- 卡片通用：
  - 背景：白色或指定浅色
  - 边框：`1px #D7E7F8`（天气卡例外见页面规格）
  - 圆角：`10`（天气卡 `12`）
- 主按钮通用：
  - 高度：`40`
  - 圆角：`10`
  - 边框：`1px #CFE2F8`
  - 文案：`14 / Medium / #FFFFFF`

## 6. 交互行为规范
- Home：
  - `进入设置` -> Settings
  - `设备状态` -> Status
- Settings：
  - `← 返回`、`保存并返回` -> Home
  - `WiFi连接`行、底部`WiFi` -> WiFi Setup
  - 列表区域必须支持垂直滚动
- Status：
  - `返回首页` -> Home
  - 列表区域必须支持垂直滚动
- WiFi：
  - `扫描`刷新 AP 列表
  - 点击 AP 填充 SSID
  - 连接失败显示红色错误提示条
  - `取消`返回 Settings

## 7. 验收标准
- 模拟器页面与 Figma 对比通过（1:1，允许仅亚像素级误差）。
- 中文字体完整、无乱码、无缺字。
- 页面跳转、滚动、WiFi交互与错误提示符合设计逻辑。
- 移植到 ESP32 后编译通过、烧录通过、实机交互通过。
